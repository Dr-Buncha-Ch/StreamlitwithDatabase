# 🧩 ใบงาน: ปรับโปรแกรม Task Manager จาก Memory ไปสู่ฐานข้อมูล SQLite

ไฟล์ที่เกี่ยวข้อง:
- `appv2CSSjavaScript.py` → โปรแกรมเดิม (เก็บข้อมูลใน session_state)
- `appv3_database.py` → เฉลยฉบับเต็ม (ใช้สอน/ตรวจคำตอบ)
- `appv3_exercise.py` → ไฟล์ให้นักเรียนฝึกเติมโค้ด (ฟังก์ชันฐานข้อมูลว่างไว้)

---

## ส่วนที่ 1: อธิบายการเปลี่ยนแปลง (ครูใช้สอนก่อนเริ่มฝึก)

### 1.1 โครงสร้างโปรแกรม — ไม่เปลี่ยน
ฟังก์ชันเดิมทั้งหมด (`home_page`, `add_task`, `show_tasks`, `delete_task`, `footer`, `main`)
**ยังอยู่ครบ ชื่อเดิม หน้าตาเมนูเดิม** ตั้งใจให้นักเรียนเห็นว่า
การเปลี่ยนจาก memory ไปฐานข้อมูล ไม่ต้องรื้อโปรแกรมทั้งหมด
แค่เปลี่ยน "ที่เก็บข้อมูล" เท่านั้น

### 1.2 สิ่งที่เพิ่มใหม่: ฟังก์ชันฐานข้อมูล 5 ตัว

| ฟังก์ชันเดิม (v2)                          | ฟังก์ชันใหม่ (v3)                          | หน้าที่ |
|---|---|---|
| `init_data()` (สร้าง list ว่างใน session_state) | `init_db()` | สร้างตาราง `tasks` ในไฟล์ `tasks.db` ถ้ายังไม่มี |
| `st.session_state.tasks.append(...)` | `add_task_db(task, status)` | เพิ่มแถวข้อมูลใหม่ |
| `pd.DataFrame(st.session_state.tasks)` | `get_all_tasks_db()` | ดึงข้อมูลทั้งหมดออกมาเป็น DataFrame |
| `st.session_state.tasks.pop(index)` | `delete_task_db(task_id)` | ลบข้อมูลตาม `id` จริงในฐานข้อมูล (**ไม่ใช่ index ในลิสต์**) |
| — (ไม่มีของเดิม) | `get_connection()` | เปิดการเชื่อมต่อฐานข้อมูล ใช้ซ้ำในทุกฟังก์ชันด้านบน |

**จุดสำคัญที่ต้องเน้นกับนักเรียน:** เดิมตอนลบข้อมูล เราใช้ "ตำแหน่งในลิสต์" (index)
แต่พอใช้ฐานข้อมูล เราต้องลบด้วย "id" ที่ฐานข้อมูลสร้างให้อัตโนมัติ (AUTOINCREMENT)
เพราะลำดับที่แสดงบนตารางกับ id จริงในฐานข้อมูลอาจไม่ตรงกันหลังมีการลบไปแล้วบางรายการ

### 1.3 จุดที่แก้ในฟังก์ชันเดิม (แก้เฉพาะจุด ไม่แก้ทั้งฟังก์ชัน)

- ใน `add_task()` : บรรทัด `st.session_state.tasks.append({...})` → เปลี่ยนเป็น `add_task_db(task, status)`
- ใน `show_tasks()` : บรรทัด `df = pd.DataFrame(st.session_state.tasks)` → เปลี่ยนเป็น `df = get_all_tasks_db()`
- ใน `delete_task()` : เปลี่ยนจากอ่าน `st.session_state.tasks` มาอ่าน `get_all_tasks_db()` และเปลี่ยนการลบจาก `.pop(index)` เป็น `delete_task_db(ids[selected])`
- ใน `main()` : บรรทัด `init_data()` → เปลี่ยนเป็น `init_db()`

### 1.4 การปรับ CSS ให้ดูมืออาชีพขึ้น (จุดที่แก้)

| จุดที่แก้ | เดิม | ใหม่ | เหตุผล |
|---|---|---|---|
| ฟอนต์ | ไม่ได้กำหนด (ใช้ฟอนต์ default ของเบราว์เซอร์) | เพิ่ม `@import` ฟอนต์ Poppins จาก Google Fonts | ฟอนต์สวย อ่านง่าย ดูเป็นมืออาชีพ |
| พื้นหลัง Sidebar | ใช้รูปภาพอย่างเดียว | เพิ่ม `linear-gradient` สีเข้มทับรูปก่อน | ตัวหนังสือสีขาวอ่านง่ายขึ้น ไม่จมไปกับรูป |
| ปุ่ม | สีพื้นเรียบสีเดียว | ใช้ `linear-gradient` ไล่สี + เงา (`box-shadow`) + เอฟเฟกต์ `transition` | ดูมีมิติ ไม่แบนเหมือนเดิม |
| ปุ่มตอน hover | เปลี่ยนสีอย่างเดียว | เพิ่ม `transform: translateY(-1px)` ให้ปุ่มลอยขึ้นเล็กน้อย | รู้สึกตอบสนอง (interactive) มากขึ้น |
| ช่องกรอกข้อความ | ไม่มีสไตล์ตอน focus | เพิ่มกรอบสีน้ำเงินตอนคลิก (`:focus`) | ผู้ใช้รู้ว่ากำลังพิมพ์อยู่ช่องไหน |
| หัวข้อ h2 | สีเขียวอย่างเดียว | เพิ่มเส้นใต้บาง ๆ (`border-bottom`) | แบ่งส่วนหัวข้อให้ชัดเจนขึ้น |
| ตารางข้อมูล | ขอบมนอย่างเดียว | เพิ่มเงา (`box-shadow`) | ตารางดูลอยขึ้นมาจากพื้นหลัง ไม่แบนราบ |

## ส่วนที่ 2: แบบฝึกหัด — ประกอบร่างโค้ดฐานข้อมูล

เปิดไฟล์ `appv3_exercise.py` จะเห็นฟังก์ชัน 5 ตัวที่มีแต่คอมเมนต์ `TODO` และคำว่า `pass`
หน้าที่ของนักเรียนคือ **เลือกชิ้นส่วนโค้ดด้านล่าง** (สลับลำดับกันไว้แล้ว) มาวางแทน `pass`
ให้ถูกฟังก์ชัน และ **เรียงลำดับให้ถูกต้อง**

> ⚠️ กติกา: ชิ้นส่วนโค้ดด้านล่างนี้ถูก "สลับลำดับ" แบบตั้งใจ และมีบางชิ้น
> ใช้ซ้ำได้ในหลายฟังก์ชัน (เช่น `conn.commit()`, `conn.close()`)
> นักเรียนต้องคิดเองว่าแต่ละฟังก์ชันต้องใช้ชิ้นไหนบ้าง และเรียงก่อน-หลังอย่างไร

### 🧩 คลังชิ้นส่วนโค้ด (สลับลำดับแล้ว)

```
E) conn = get_connection()

A) return sqlite3.connect(DB_NAME, check_same_thread=False)

H) conn.commit()

C) cursor = conn.cursor()

J) cursor.execute("""
       CREATE TABLE IF NOT EXISTS tasks (
           id INTEGER PRIMARY KEY AUTOINCREMENT,
           task TEXT NOT NULL,
           status TEXT NOT NULL
       )
   """)

F) cursor.execute(
       "INSERT INTO tasks (task, status) VALUES (?, ?)",
       (task, status)
   )

D) cursor.execute("DELETE FROM tasks WHERE id = ?", (task_id,))

B) df = pd.read_sql_query("SELECT * FROM tasks", conn)

I) return df

G) conn.close()
```

### โจทย์: จับคู่ชิ้นส่วนกับฟังก์ชันให้ถูกต้อง

1. **`get_connection()`** ต้องการชิ้นส่วนกี่ชิ้น? ชิ้นไหนบ้าง?
2. **`init_db()`** ต้องเรียงกี่ชิ้น และต้องใช้ชิ้นที่มีคำว่า `CREATE TABLE` ตรงไหนของลำดับ?
3. **`add_task_db()`** ต่างจาก `init_db()` ตรงไหน (ใช้ชิ้นเหมือนกันกี่ชิ้น ต่างกันชิ้นไหน)?
4. **`get_all_tasks_db()`** ทำไมถึง**ไม่ต้อง**ใช้ `cursor = conn.cursor()`?
   (คำใบ้: `pd.read_sql_query` รับ `conn` ตรง ๆ ได้เลย ไม่ต้องผ่าน cursor)
5. **`delete_task_db()`** เหมือนกับ `add_task_db()` ตรงไหน และต่างกันตรงไหน?

> 💡 เฉลยลำดับที่ถูกต้องสามารถเปิดดูได้จากไฟล์ `appv3_database.py`
> แต่ให้นักเรียนลองประกอบเองก่อน แล้วรัน `streamlit run appv3_exercise.py`
> เพื่อดูว่าเกิด error ตรงไหนไหม เป็นวิธีฝึกอ่าน error message ไปในตัว
